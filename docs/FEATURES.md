# LAN-ClipSync 功能介绍

**局域网剪切板同步助手** — 在多台 Windows 电脑之间，像使用本机剪切板一样同步纯文本内容。

---

## 它能做什么

| 功能 | 说明 |
|------|------|
| 实时文本同步 | 在一台电脑上复制文字，已配置的对端会自动写入剪切板，可直接粘贴 |
| 无长度限制 | 与 Windows 系统剪切板一致，**不截断**长文本（段落、代码、日志均可） |
| 仅纯文本 | 只处理 Unicode/ANSI 文本；文件、图片等非文本内容会被忽略，避免误传 |
| 局域网专用 | 所有通信仅在 LAN 内完成，**不访问互联网** |
| 后台托盘运行 | 启动后驻留系统托盘，不占用任务栏；双击或右键可打开设置 |
| 二维码生成 | 内置离线二维码工具，可将本机地址等文本生成二维码，方便手机扫码或截图分享 |

---

## 适用场景

- 办公桌上 **两台或多台 PC** 协作：主屏复制、副屏粘贴
- 开发机与测试机之间 **快速传递命令、配置、片段代码**
- 虚拟机与宿主机、远程桌面与本地之间 **传文本**（需网络路由可达）
- 企业内网多 VLAN 环境：通过 **手动配置 IP/计算机名**，不依赖 UDP 广播
- 不方便口头报 IP 时：用 **生成二维码** 展示本机监听地址，对方扫码后配置对端

## 使用流程

### 1. 安装与启动

1. 在需要同步的每台 Windows 10/11（64 位）电脑上运行 `LAN-ClipSync.exe`
2. 首次启动可能弹出 **Windows 防火墙** 提示，请允许 **专用/局域网** 访问
3. 程序在托盘显示图标（产品名：**局域网剪切板同步助手**）

### 2. 配置对端（双方都要做）

打开 **对端设置**（托盘图标 **双击** 或 **右键 → 对端设置**）：

1. 将窗口顶部的 **本机监听地址**（如 `192.168.1.10:52341`）发给对方
2. 在列表中添加对方的 **`IP:端口`** 或 **计算机名**（可省略端口，程序会自动探测）
3. **双方必须互相添加** 对方地址，单向配置无法同步

配置保存在 exe 同目录的 `LAN-ClipSync-peers.json`，重启后自动沿用。

### 3. 同步文本

在任意一台已互联的电脑上 **Ctrl+C 复制** 纯文本，对端剪切板会自动更新，在对端 **Ctrl+V 粘贴** 即可。

### 4. 二维码生成（分享连接信息）

除手动复制 `IP:端口` 外，程序内置 **离线二维码生成** 工具，便于把本机监听地址发给同事或用手机扫码查看。

**打开方式**：托盘图标 **右键 → 生成二维码**

**功能说明**：

| 能力 | 说明 |
|------|------|
| 实时生成 | 上方文本框输入或编辑内容，下方二维码约 350ms 内自动刷新 |
| 自动填充 | 打开窗口时会尝试读取当前剪切板文本（便于直接粘贴本机地址） |
| 高 DPI 适配 | 二维码显示区域随窗口大小与屏幕 DPI 自动缩放，保证扫码清晰 |
| 纯离线 | 不联网、不上传；仅在本地根据文本绘制二维码 |
| 典型用途 | 将 `192.168.1.10:52341`、计算机名地址或多段 `IPv4;IPv6;计算机名` 格式生成二维码，对方扫码后手动填入对端设置 |

**使用建议**：

1. 在对端设置窗口 **复制本机监听地址**
2. 打开 **生成二维码**，地址会自动或手动粘贴到文本框
3. 将屏幕上的二维码展示给对方扫码，或截图发送
4. 对方将解码得到的地址添加到 **对端设置** 即可

**容量说明**：二维码遵循 QR Code 标准，单码可编码的文本量有限（与字符集、纠错级别有关；本工具使用 **Medium** 纠错）。极长文本可能无法生成，此时请改用直接复制地址字符串。这与剪切板同步的「无长度限制」是两套独立能力。

**第三方组件**：二维码 **编码** 能力基于 [Project Nayuki](https://www.nayuki.io/page/qr-code-generator-library) 开源的 **QR Code generator library**（MIT License），源码位于 `src/qrcodegen.cpp`、`src/qrcodegen.hpp`。感谢 Nayuki 提供高质量、零依赖的二维码创建实现。

### 5. 其他操作

| 操作 | 方式 |
|------|------|
| 刷新对端连接状态 | 对端设置窗口内 **刷新** |
| 删除对端 | **双击** 已连接/未连接的对端行 |
| 自动探测端口 | **双击** 未连接行，或输入 IP/计算机名后回车 |
| 生成二维码 | 托盘 **右键 → 生成二维码** |
| 退出程序 | 托盘 **右键 → 退出** |

## 网络与连接特性

- **协议**：TCP，帧格式为「4 字节长度 + JSON 正文」
- **连接方式**：点对点出站连接，便于跨网段；**不使用 UDP 广播**
- **监听端口**：首次启动优先尝试约定端口（12123、12345、12306），否则在 45000–65000 随机选取并写入配置
- **双栈支持**：同时支持 **IPv4 与 IPv6** 监听与连接
- **计算机名**：配置可保存计算机名，域环境 IP 变化后仍可解析（需 DNS/NetBIOS 可达）
- **本机地址格式**：复制时可得到 `IPv4:端口;[IPv6]:端口;计算机名:端口`，粘贴时整行解析

## 安全与隐私说明

- 当前版本在局域网内 **明文传输** 文本，同一网段内具备嗅探能力的设备理论上可读取内容
- 请勿在不可信网络中使用；敏感信息建议配合 VPN 或后续版本的加密扩展
- 程序 **不会** 上传数据到公网服务器
- 仅同步文本，**不会** 同步文件路径、图片或剪切板中的其他格式

## 限制与已知行为

| 项目 | 说明 |
|------|------|
| 平台 | 仅 **Windows 10/11 x64** |
| 内容类型 | 仅 **纯文本**（`CF_UNICODETEXT` / `CF_TEXT`） |
| 文本长度 | **无 artificial 限制**；极大文本受系统内存与网络传输时间影响 |
| 同步方向 | 本地复制 → 发往所有已配置对端；收到网络文本 → 写入本地剪切板 |
| 防回环 | 程序写入的来自网络的内容不会再次广播，避免 A↔B 无限循环 |
| 多对端 | 可同时配置多个对端，一次复制会发送给全部已配置节点 |
| 二维码 | 单码文本容量受 QR 标准限制（非剪切板同步长度）；编码基于 Nayuki 库，见下文 |

## 日志与排错

- 运行日志：exe 同目录 **`LAN-ClipSync.log`**
- 无法同步时请检查：
  1. 双方是否 **互相** 添加了对端
  2. Windows 防火墙是否放行 TCP 入站
  3. 两端 IP/计算机名是否在同一网络或路由可达
  4. 复制的是否为 **纯文本**（非文件、非图片）

更详细的编译与故障排除见 **[BUILD.md](../BUILD.md)**。
